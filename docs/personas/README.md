# Finance Tracker Personas

This directory contains persona definitions for testing, role-playing, and understanding different user types of Finance Tracker.

## Available Personas

1. **[Sarah Chen](./sarah-chen.md)** - Budget-Conscious Millennial
   - Young professional learning to manage money
   - Wants to cut spending and save for big goals
   - Non-technical, visual learner

2. **[Marcus Rodriguez](./marcus-rodriguez.md)** - Freelance Business Owner
   - Consultant juggling business and personal finances
   - Needs tax organization and expense tracking
   - Moderately technical, efficiency-focused

3. **[Alex Kim](./alex-kim.md)** - Privacy-Focused Data Engineer
   - Technical professional who values data sovereignty
   - Wants automation and self-hosted solutions
   - Highly technical, customization-oriented

## Usage

Each persona file contains:
- **Name**: The persona's identity
- **Short Description**: Background and characteristics
- **System Prompt**: LLM prompt for role-playing this persona

### Using Personas for Testing

These personas can be used to:
- Test user flows from different perspectives
- Generate realistic questions and feedback
- Create training documentation
- Simulate user interviews
- Test AI categorization prompts

### Using System Prompts

The system prompts can be used with LLMs to:
1. Role-play as the persona in conversations
2. Generate persona-specific feature requests
3. Create realistic test scenarios
4. Validate documentation clarity
5. Test onboarding flows

**Example**:
```
System: [Copy system prompt from persona file]
User: How would you categorize your coffee shop purchases?
Assistant: [Responds as the persona]
```
